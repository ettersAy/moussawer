# Reflection Notes ‚Äî 2026-05-05

## Time Wasters

- Playwright MCP browser testing was dead on arrival ‚Äî Chrome symlink `/opt/google/chrome/chrome` pointed to `/tmp/` which doesn't survive reboots. Spent 20min diagnosing, attempting sudo-less fixes, before pivoting to API-only testing via curl
- Supabase free-tier DB was unreachable (`FATAL: Tenant or user not found`). No local fallback documented. Had to discover Docker was available, pull postgres image, and reconfigure `.env`
- `pkill -f "tsx watch"` killed the shell process itself (exit 144). Shell pattern matched the running bash session. Safe alternative: `kill $(lsof -ti:PORT)`
- `status` is a zsh reserved word. Using it as a variable name in `for` loops causes silent failures. Use different names in zsh scripts
- `curl` not found in non-interactive zsh when called as bare `curl`. Use full path `/usr/bin/curl` or ensure PATH is set

## Missing Project Knowledge

- No CLAUDE.md or .clinerules exists ‚Äî zero AI guidance for this repo
- `.env.example` shows Supabase pooler connection string but no hint that free tier pauses after 1 week of inactivity or how to switch to local
- Service dependency (PostgreSQL only) is undocumented ‚Äî had to infer from Prisma schema
- Dev vs prod runtime difference: `tsx watch` in dev, compiled `dist-server/` in prod. Not obvious from scripts alone
- Prisma deprecated config (`package.json#prisma`) generates warning on every command ‚Äî knowing this is expected would avoid re-investigation each time
- Booking availability system uses photographer's timezone (e.g. America/Toronto) but accepts UTC timestamps ‚Äî caused confusion when bookings were rejected with `SLOT_UNAVAILABLE`

## Architecture Discoveries

- 20 Prisma models, 33 API endpoints, 11 frontend routes ‚Äî medium-sized MVP
- Route ‚Üí Include ‚Üí Resource 3-layer pattern is consistent across all endpoints
- `server/routes/helpers.ts` has shared access-check functions worth reusing
- `server/routes/includes.ts` centralizes Prisma include objects
- Tags stored as comma-separated string in DB, not a relation ‚Äî intentional MVP simplification
- Availability service has duplicate query logic between `availabilityForDate` and `assertBookableSlot` ‚Äî known tech debt

## Repeated Operations Worth Automating

- Killing/restarting dev servers happened 6+ times across 3 rounds
- Re-login to get fresh JWT tokens for each test batch ‚Äî tokens expire in 14d but script re-fetched each time
- Same curl test commands repeated across rounds ‚Äî a test smoke script would have saved 40% of the time
- Database connection testing: `psql`, `nslookup`, `prisma db push` trio repeated
- `npx prisma db push` + `npx prisma generate` always run as a pair after schema changes

## Incorrect Assumptions

- Assumed Playwright MCP would work out of the box ‚Äî should verify browser availability first
- Assumed Supabase cloud DB would be reachable ‚Äî should test connectivity before committing to workflow
- Assumed sudo would work for systemctl ‚Äî user had no passwordless sudo
- Assumed `localStorage` JWT was the only auth concern ‚Äî missed that the whole token flow lacks refresh/blacklist

## What Would Have Helped

- A pre-flight checklist: DB reachable? Browser available? Ports free? Sudo available?
- A single smoke-test script that exercises all endpoints and reports pass/fail
- A `CLAUDE.md` documenting: tech stack, how to run, known issues (Supabase pausing, timezone gotcha), key file locations
- A restart script: `kill $(lsof -ti:4000) $(lsof -ti:5173); npm run dev`
